# 智能花卉识别与科普系统需求规格说明书

**版本号**：V1.0

**日期**：2026年1月17日

------

## 1. 引言

### 1.1 项目背景

随着生活水平提升，花卉成为现代生活重要组成部分。传统花卉识别依赖人工查阅或简单图像识别，存在效率低、信息分散、准确率不足等问题。本系统通过集成多模态大模型API（如deepseek，通义千问），提供智能识别与科普服务。

### 1.2 项目目标

- **核心目标**：解决花卉识别效率低、信息分散、科普内容不足的问题
- **用户价值**：提供一站式花卉识别与科普服务
- **行业价值**：赋能植物园、电商平台等合作伙伴

------

## 2. 总体描述

### 2.1 功能范围

| 模块          | 功能点                                       | 优先级 |
| ------------- | -------------------------------------------- | ------ |
| 图像识别      | 单张图片上传识别，返回名称、科属、特征描述   | P0     |
| 科普信息生成  | 基于识别结果生成养护方法、花语、文化内涵     | P0     |
| 智能问答      | 结合图片与自然语言提问生成回答，支持多轮对话 | P1     |
| 知识库管理    | 后台增删改查花卉信息，支持关键词检索         | P1     |
| 响应式Web界面 | 适配PC/移动端，支持图片上传与结果展示        | P0     |

### 2.2 非功能需求

| 类别         | 要求                                             |
| ------------ | ------------------------------------------------ |
| **性能**     | 识别响应时间 ≤3秒，准确率 ≥90%                   |
| **安全性**   | API密钥加密存储，支持HTTPS                       |
| **兼容性**   | 支持Chrome/Edge/微信浏览器，适配主流移动端分辨率 |
| **可扩展性** | 支持多模态大模型动态切换（如deepseek、通义千问） |

------

## 3. 外部接口

### 3.1 用户接口

| 接口类型         | 功能描述                                 |
| ---------------- | ---------------------------------------- |
| 图片上传接口     | 支持JPG/PNG格式，单张图片大小≤5MB        |
| 识别结果查询接口 | 通过Image ID获取花卉名称、科属、特征描述 |
| 科普信息生成接口 | 输入花卉ID生成养护方法、花语等内容       |
| 智能问答接口     | 结合图片ID与自然语言问题生成回答         |

### 3.2 系统接口

| 接口类型  | 目标系统                                |
| --------- | --------------------------------------- |
| 大模型API | deepseek、通义千问等                    |
| 数据库    | MySQL存储用户信息、识别历史、知识库数据 |

------

## 4. 系统特性

### 4.1 核心功能

#### 4.1.1 图像识别

- **输入**：用户上传的单张花卉图片
- **处理**：调用多模态大模型API进行图像识别
- **输出**：JSON格式识别结果（名称/科属/特征）

#### 4.1.2 科普信息生成

- **输入**：花卉识别结果
- **处理**：调用大模型API生成养护方法、花语等内容
- **输出**：结构化科普信息（Markdown格式）

#### 4.1.3 智能问答

- **输入**：用户自然语言问题（如"如何防治蚜虫？"）
- **处理**：结合图片特征与问题生成回答
- **输出**：自然语言回答（支持多轮对话）

## 5. 其他需求

### 5.1 数据需求

| 数据表     | 字段示例                                                    |
| ---------- | ----------------------------------------------------------- |
| 用户表     | user_id, username, email, registration_date                 |
| 知识库表   | plant_id, name, family, care_instructions, cultural_meaning |
| 识别记录表 | record_id, image_url, plant_id, user_id, create_time        |

### 5.2 非功能性约束

- **API调用限制**：每分钟≤100次（需申请大模型API白名单）
- **日志记录**：记录用户操作日志与系统异常日志
- **备份策略**：每日凌晨自动备份数据库
