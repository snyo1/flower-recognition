## **一、系统架构总览**

### 1.1 架构设计原则

- 

  **前后端分离**：前端专注交互体验，后端提供标准化API，通过Axios通信。

- 

  **模块化设计**：按功能拆分独立模块（识别、科普、问答、知识库），降低耦合。

- 

  **弹性扩展**：支持大模型API动态切换（DeepSeek/通义千问）、数据库读写分离、异步任务扩容。

- 

  **安全合规**：API密钥加密存储、HTTPS传输、用户输入校验、操作日志审计。

## **三、核心模块设计**

### 3.1 前端模块

#### 3.1.2 状态管理（Pinia）

- **store/modules/user.js**：用户信息（ID/昵称/角色）、登录状态、Token管理。

- **store/modules/identify.js**：当前识别结果（图片URL/名称/科属）、识别状态（加载中/成功/失败）。

- **store/modules/chat.js**：对话历史（问题/回答/时间戳）、上下文窗口（最近5轮）。

### 3.2 后端模块

#### 3.2.1 API服务（FastAPI）

| **API类别**    | **核心接口**                     | **方法**            | **功能描述**                                                 |
| -------------- | -------------------------------- | ------------------- | ------------------------------------------------------------ |
| **图片上传**   | `/api/v1/upload/image`           | POST                | 接收用户上传图片，存储至MinIO，返回图片URL、大小、格式、预览URL |

#### 3.2.2 大模型集成模块（LangChain）

- **API封装**：使用`langchain_community.chat_models`调用DeepSeek/通义千问API，统一接口格式。

- **提示词模板**：

  ```python

  IDENTIFY_PROMPT = PromptTemplate(

      - family: 科属分类（如“蔷薇科 蔷薇属”）

  )  
  
  # 科普生成提示词模板  
  SCIENCE_PROMPT = PromptTemplate(  
      input_variables=["name", "family", "features"],

      ## 养护方法（浇水/施肥/光照）  
      ## 病虫害防治
  ```

- **上下文管理**：用Redis存储用户对话历史（Key: `chat:{user_id}:{session_id}`，Value: 对话列表），支持多轮关联。

#### 3.2.3 知识库管理模块

- **数据结构**（MySQL表`plants`）：

  ```sql
  CREATE TABLE plants (  
    id INT PRIMARY KEY AUTO_INCREMENT,  
    name VARCHAR(255) NOT NULL COMMENT '花卉名称',  
    family VARCHAR(255) COMMENT '科属分类',  
    features TEXT COMMENT '特征描述',  
    growth_habit TEXT COMMENT '生长习性',  
    care_method TEXT COMMENT '养护方法',  
    pest_control TEXT COMMENT '病虫害防治',  
    cultural_meaning TEXT COMMENT '花语文化内涵',  
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,  
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,  
    FULLTEXT INDEX idx_search (name, family, features)  -- 全文索引支持关键词搜索  
  ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
  ```
  
- **检索逻辑**：用户输入关键词，通过`MATCH(name, family, features) AGAINST('关键词')`实现模糊匹配。

## **六、关键技术实现细节**

### 6.1 多模态API调用流程

```python
# 基于LangChain的大模型调用示例（识别功能）  
from langchain_community.chat_models import ChatDeepSeek  
from langchain.prompts import ChatPromptTemplate  

def identify_flower(image_url: str):  
    # 1. 初始化大模型客户端（API密钥从环境变量读取）  
    llm = ChatDeepSeek(  
        api_key=os.getenv("DEEPSEEK_API_KEY"),  
        model="deepseek-vl-7b-chat"  # 多模态模型  
    )  

    # 2. 构造提示词  
    prompt = ChatPromptTemplate.from_template(  
        "你是一个花卉识别专家，请分析图片{image_url}中的花卉，返回JSON格式结果："  
        "{{'name': '名称', 'family': '科属', 'features': ['特征1', '特征2'], 'confidence': 95}}"  
    )  

    # 3. 调用API（传入图片URL，LangChain自动处理多模态输入）  
    chain = prompt | llm  
    response = chain.invoke({"image_url": image_url})  

    # 4. 解析结果（JSON格式）  
    return json.loads(response.content)
```

### 6.2 异步任务处理（Celery）

- **场景**：大模型API调用耗时较长（3-5秒），通过Celery异步执行，避免阻塞API响应。

- **示例任务**：

  ```python
  # tasks.py  
  from celery import Celery  
  
  app = Celery("tasks", broker="redis://redis:6379/0")  
  
  @app.task  
  def async_identify_flower(image_url: str, user_id: int):  
      result = identify_flower(image_url)  # 调用上述识别函数  
      # 保存识别记录至数据库  
      save_identify_record(user_id, image_url, result)  
      return result
  ```

## **七、测试与监控**

| **测试类型** | **工具**         | **覆盖范围**                                                 |
| ------------ | ---------------- | ------------------------------------------------------------ |
| 单元测试     | Pytest           | 后端核心函数（识别/科普生成/知识库CRUD）、前端组件（上传/预览） |

### 7.2 监控与日志

- **日志**：用`loguru`记录操作日志（用户行为）、异常日志（API调用失败），输出至ELK Stack（Elasticsearch存储、Kibana可视化）。

- **监控**：Prometheus采集系统指标（CPU/内存/请求量），Grafana配置仪表盘（识别成功率、问答响应时间、知识库检索耗时）。

## **八、安全设计**

- **API安全**：JWT令牌认证（用户登录后颁发，有效期2小时），敏感接口（如知识库管理）校验管理员角色。

- **数据安全**：用户密码bcrypt哈希存储，API密钥AES-256加密后存配置文件，数据库定期备份（每日全量+binlog增量）。

- **输入校验**：前端用Element Plus表单验证，后端用Pydantic模型校验（如图片格式/大小、用户输入问题长度）。

**备注**：本架构预留**扩展接口**，支持未来新增功能（如批量识别、AR识别、用户社交分享），可通过动态加载模块或插件化设计实现。
